# 競合note分析

## 分析フレームワーク

### チェック項目
- [ ] タイトルの型（数字型・疑問型・How to型など）
- [ ] 文字数・見出し数
- [ ] サムネイルデザインの傾向
- [ ] スキ数・コメント数
- [ ] 有料 or 無料設定
- [ ] 投稿頻度・曜日・時間帯

## バズ記事の共通パターン

### タイトルの傾向
1. 数字を入れる →「3つの方法」「30日で〇〇」
2. 読者を主語にする → 「あなたが〇〇できない理由」
3. 逆説・意外性 → 「〇〇してはいけない理由」
4. 具体的な数字の実績 → 「フォロワー0から1,000人になった方法」

### 構成の傾向
- リード文：問題提起 → 共感 → 解決策の予告
- 本文：PREP法（結論→理由→具体例→結論）
- 締め：行動促進 + フォロー促進

## 競合が手薄なポジション（ブルーオーシャン）

- 特定のニッチ職種 × note活用
- 失敗談に特化したコンテンツ
- ビジュアル（図解・表）を多用した解説記事
