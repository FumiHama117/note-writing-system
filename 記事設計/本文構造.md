# 本文構造

## 基本原則

### PREP法（各セクションの基本構造）
1. **P**oint（結論）: 最初に主張を述べる
2. **R**eason（理由）: なぜそう言えるのか
3. **E**xample（具体例）: 実例・数字・体験談
4. **P**oint（結論の再確認）: まとめて強調

## 本文の品質基準

### 必ず含める要素
- **具体的な数字**: 「多くの人」ではなく「73%の人」
- **実体験**: 「私が実際に〇〇した時」
- **固有名詞**: ツール名、サービス名、人名
- **比較**: 「〇〇より△△の方が効果的な理由」

### 読みやすさの工夫
- 1段落は3〜5行以内
- 箇条書きを適切に活用（3〜5項目）
- 重要な単語は**太字**で強調
- 長い説明には表を活用

## セクションの書き方テンプレート

### 方法・手順を説明するセクション
```markdown
## [セクションタイトル]

### [小見出し1]
[結論を1文で]

その理由は[理由]です。実際に私が[体験談]したところ、[具体的な結果]という結果になりました。

**実践ステップ**:
1. まず〇〇をする
2. 次に△△を確認する
3. 最後に□□を設定する

> **ポイント**: [読者が見落としがちな重要点]

### [小見出し2]
[同じ構造で続ける]
```

### 比較・解説するセクション
```markdown
## [セクションタイトル]

### [小見出し1]

| 比較項目 | 方法A | 方法B |
|----------|-------|-------|
| 効果 | ★★★ | ★★★★★ |
| 難易度 | 高い | 低い |
| 時間 | 30分/日 | 10分/日 |

[表の補足説明と推奨理由]
```

## 締めの書き方

### 締め文の構成（必須）
1. **本文の要約**: 3点以内で整理
2. **読者への問いかけ**: 「あなたはどれから始めますか？」
3. **次のアクション**: 明確な行動を1つだけ提示
4. **フォロー・スキ促進**: 自然な形で呼びかけ

### 締め文テンプレート
```
今回は〇〇について解説しました。

ポイントをまとめると：
- 〇〇（最重要ポイント）
- △△（次に大切なこと）
- □□（見落としがちなこと）

まず最初にやることは[1つだけの行動]です。今日中に試してみてください。

この記事が参考になったら、スキ・フォローをいただけると励みになります。次回は〇〇について書く予定です。
```
