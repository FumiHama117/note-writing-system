# note記事執筆システム 使い方マニュアル

## このシステムでできること

Claude Codeに話しかけるだけで、noteに投稿できる高品質な記事を自動生成できます。  
タイトル案の提示から本文の構成・執筆・チェックまで、一連の流れをClaude Codeが担います。

---

## 前提：動作環境

- **Claude Code** がインストールされていること
- このリポジトリ（`note-writing-system/`）をワーキングディレクトリとして開いていること

Claude Codeは起動時に `CLAUDE.md` を自動で読み込み、note記事執筆AIライターとして動作します。

---

## 基本的な使い方

### ステップ1：テーマを伝える

Claude Codeに記事のテーマや方向性を伝えます。

**例文**
```
「note初心者向けに、フォロワーを増やす方法について記事を書いて」
「副業でnoteを収益化する方法をまとめた記事を作成して」
「有料記事の価格設定について、体験談ベースで書いて」
```

追加情報を渡すとより精度が上がります。

```
「ターゲットは会社員の副業希望者。
 キーワードは『note 副業 始め方』。
 有料記事として4,000字程度で書いて」
```

---

### ステップ2：タイトル案を選ぶ

Claude Codeが `/記事設計/タイトル.md` の型を参照し、タイトル案を3つ提示します。

```
1. 「副業初心者がnoteで月3万円を稼ぐ5つのステップ」
2. 「noteを始めて6ヶ月で収益化できた私がやったこと全部」
3. 「なぜあなたのnoteは売れないのか？副業で稼げる人との3つの違い」
```

好みのタイトルを選ぶか、修正を指示してください。

---

### ステップ3：記事を生成する

タイトルが決まると、Claude Codeが次の順番で記事を組み立てます。

| 順番 | 参照ファイル | 生成内容 |
|------|-------------|---------|
| 1 | `戦略/読者リサーチ.md` | ターゲット読者の確認 |
| 2 | `戦略/ニーズ把握.md` | ペインポイントの特定 |
| 3 | `記事設計/リード文.md` | 最初の3行（リード文） |
| 4 | `記事設計/見出し構成.md` | H2/H3の見出し構成 |
| 5 | `記事設計/本文構造.md` | 本文全体 |
| 6 | `templates/review_checklist.md` | 投稿前チェック |

---

### ステップ4：記事を保存する

生成された記事を `outputs/` フォルダに保存するよう指示します。

```
「outputs/ に保存して」
「outputs/2024-01-15_note副業始め方.md として保存して」
```

---

## よく使う指示フレーズ集

### 記事を書かせる

```
〇〇について記事を書いて
〇〇をテーマにnote記事を作成して
〇〇向けの記事を3,000字で書いて
```

### 修正・調整させる

```
タイトルをもっとキャッチーにして
リード文を書き直して、もっと共感を引き出す内容に
〇〇のセクションを具体例を増やして書き直して
全体のトーンをもう少し親しみやすくして
文字数が少ないので、〇〇の部分を膨らませて
```

### チェックさせる

```
投稿前チェックリストで確認して
SEO的に問題ないか確認して
ハッシュタグを5つ提案して
```

### 個別パーツだけ生成させる

```
タイトル案だけ5つ出して
リード文だけ書いて
見出し構成だけ先に見せて
締めの一文を3パターン書いて
```

## ファイル構成と役割

```
note-writing-system/
│
├── CLAUDE.md                   ← Claude Codeへの指示書（自動読み込み）
│
├── 戦略/                       ← 記事の方向性を決める知識ベース
│   ├── 読者リサーチ.md          ペルソナ・読者像
│   ├── ニーズ把握.md            ペインポイント・求めていること
│   ├── 競合note分析.md          バズ記事のパターン分析
│   ├── ポジショニング.md         差別化の軸・ブランドボイス
│   └── 成功事例研究.md          収益化パターン・成功事例
│
├── 記事設計/                    ← 記事の各パーツのテンプレ・型集
│   ├── タイトル.md              バズるタイトルの6型
│   ├── 見出し構成.md            H2/H3の黄金比・3パターン
│   ├── リード文.md              最初の3行テンプレ（3型）
│   ├── 本文構造.md              PREP法・セクションテンプレ
│   ├── 内部リンク.md            リンク設置戦略
│   ├── サムネイル.md             デザイン法則・ツール
│   └── 締めの一文.md            締め文4型
│
├── プラットフォーム理解/         ← noteの仕組みを知る
│   ├── noteアルゴリズム.md      露出を増やす要因・投稿タイミング
│   ├── SEO評価.md              キーワード戦略
│   └── ハッシュタグ運用.md      タグ選定の組み合わせ
│
├── templates/                  ← 自分で書くときのテンプレ
│   ├── article_template.md     記事執筆テンプレ（事前設計欄付き）
│   └── review_checklist.md     投稿前チェックリスト
│
├── outputs/                    ← 生成した記事の保存先
│
└── docs/
    └── 使い方マニュアル.md      ← このファイル
```

## 知識ベースのカスタマイズ

各MDファイルを編集することで、Claude Codeの振る舞いを自分に合わせて調整できます。

### よくあるカスタマイズ例

**自分のターゲット読者を具体化したい**
→ `戦略/読者リサーチ.md` のペルソナ欄を実際の読者像に書き換える

**特定の文体・トーンに統一したい**
→ `戦略/ポジショニング.md` のブランドボイス欄に希望を追記する

**自分がよく使うタイトルパターンを追加したい**
→ `記事設計/タイトル.md` に新しい型を追加する

**チェック項目を増やしたい**
→ `templates/review_checklist.md` に独自のチェック項目を追加する

## トラブルシューティング

### 記事の品質が低いと感じる場合

1. テーマの指示をより具体的にする（ターゲット・キーワード・文字数を明示）
2. `戦略/読者リサーチ.md` のペルソナを実際の読者に合わせて更新する
3. 生成後に「〇〇の部分をもっと具体的に」と追加指示する

### 文字数が足りない場合

```
「本文が短いので、各セクションに具体例と体験談を1つずつ追加して」
「〇〇のセクションを500字程度に膨らませて」
```

### 文体が固い・柔らかすぎる場合

```
「もっと話しかけるような文体に直して」
「もう少し丁寧な文体に統一して」
```

## outputs/ フォルダの管理方法

ファイル名の推奨形式：`YYYY-MM-DD_テーマの略称.md`

```
例:
outputs/2024-01-15_note副業始め方.md
outputs/2024-01-22_フォロワー増やし方.md
outputs/2024-02-01_有料記事価格設定.md
```

投稿済みの記事は `outputs/published/` サブフォルダに移動すると管理しやすいです。
